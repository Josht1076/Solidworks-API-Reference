Remarks

If developing a C++ application, use swAssemblyViewNewNotify2 to register for this notification.

This event is sent for each new model view created by the window splitter bar. It is also sent for each new model view created in a document when Window, New Window is selected. A Dispatch pointer to the new model view is passed, which can be used to register for IModelView events or to run the various ModelView APIs such as IModelView::GetViewHWnd.

Because FileOpenNotify2 and FileNewNotify2 are post-notifications, SOLIDWORKS has already constructed the model views and has already fired this event by the time your application receives FileOpenNotify2 or FileNewNotify2. Therefore, within your functions that handle FileOpenNotify2 and FileNewNotify2, you should programmatically traverse each model view in the newly opened or created document (IModelDoc2::EnumModelViews or IModelDoc2::GetFirstModelView or IModelDoc2::IGetFirstModelView and IModelView::GetNext or IModelView::IGetNext), and register each view for ModelView events.
